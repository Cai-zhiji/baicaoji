## 1. 快速开始

### 1.1 登录

打开应用后进入登录页，输入管理员配置的账号和密码。登录成功后 30 天内无需重复输入。

### 1.2 导航

左侧边栏包含所有功能入口：

| 菜单 | 功能 |
|------|------|
| 🏠 开方 | 创建药方、计价 |
| 👤 病人 | 病人档案管理 |
| 🌿 药材 | 药材目录维护 |
| 📋 药方 | 历史药方查询 |
| 📦 库存 | 药材进货与库存 |
| 📊 统计 | 收入与利润分析 |
| 📐 模版 | 药方模版管理 |

### 1.3 布局说明

- **顶栏**：左侧显示当前页面标题，右侧有「老年模式」切换和「登出」按钮
- **侧边栏**：点击折叠按钮可收起，节省屏幕空间
- **内容区**：主要操作区域，不同页面布局略有差异

## 2. 开方（首页）

开方页是日常使用最多的页面，用于创建药方、自动计价、保存记录。

### 2.1 选择病人

1. 在顶部搜索框输入病人**姓名或拼音首字母**
2. 从下拉列表中选择已有病人
3. 如果病人不存在，点击「新建"xxx"」快速创建（默认性别为"男"，可在病人管理中修改）

> 选择病人是可选的。不选病人即为「散客」开方，药方仍会保存。

### 2.2 添加药材

1. 在药材搜索框输入**药材名称或拼音首字母**（如输入 `dg` 可搜索「当归」）
2. 点击目标药材添加到药方
3. 在克数列输入用量（支持 0.5g 精度）
4. 单价默认为药材管理中设定的售价，不可在此修改

### 2.3 价格计算

- 每味药的小计 = 售价 × 克数
- 药方总价自动累加，显示在底部
- 价格精确到分（保留两位小数）

### 2.4 库存告警

当药材克数超过库存量时，药材名称旁会显示红色警告图标和「库存仅 Xg」提示。开方不会因库存不足被阻止，但提醒你及时进货。

### 2.5 保存药方

1. 确保每味药材的克数已填写（至少 > 0）
2. 点击底部「保存药方」按钮
3. 保存成功后：
   - 已选药材的库存自动扣减
   - 药材售价和时间被快照到药方中（后续药材调价不影响历史药方）
   - 药材列表被清空，可继续开下一方

### 2.6 清空药方

点击明细区右上角「🗑 清空」按钮，确认后清除当前所有药材。

### 2.7 加载模版

在模版搜索区直接输入模版名称，或从热门模版列表中点击加载。加载后药材和默认克数自动填入，可继续编辑。

### 2.8 保存为模版

点击明细区右上角「📋 存模版」，输入模版名称后保存。当前药方中的药材组合（含克数）将被保存，下次可直接加载。

### 2.9 再开历史药方

- 在药方记录页找到目标药方，点击「🔄 再开」
- 或在病人详情页找到药方，点击「再开」
- 药方中的药材和克数会自动填入开方页，保留原售价信息

## 3. 病人管理

路径：侧边栏 → 👤 病人

### 3.1 搜索病人

在搜索框输入姓名、拼音首字母或电话号码，实时筛选。

### 3.2 添加病人

1. 点击右上角「➕ 添加」
2. 填写姓名（必填）、性别、年龄、电话
3. 点击「添加」

> **同名归并**：如果输入的姓名已存在，系统会提示您使用已有档案，避免重复创建。

### 3.3 编辑病人

点击病人行右侧的 ✏️ 按钮，修改后保存。

### 3.4 删除病人

点击病人行右侧的 🗑 按钮，确认后删除。**病人的历史药方不会被删除**（药方中的病人关联会解除）。

### 3.5 病人详情

点击病人姓名进入详情页，可查看：
- 基本信息（姓名、性别、年龄、电话）
- 累计药方数和累计药费
- 历史药方列表（含随访记录）
- 每条药方的药材明细、复制、再开功能

### 3.6 批量导入

支持 CSV 导入。详见 [数据导入导出](#9-数据导入导出)。

## 4. 药材管理

路径：侧边栏 → 🌿 药材

### 4.1 搜索药材

- 搜索框：输入名称或拼音
- A-Z 字母索引：右侧竖条拖拽或点击字母，快速跳转到对应分组

> 未搜索时仅展示最近使用的 20 种药材。

### 4.2 添加药材

1. 点击右上角「➕ 添加」
2. 填写：
   - **名称**：药材中文名（必填）
   - **售价**：每克售价，单位元（如 0.35 表示 3.5 毛/g）
   - **成本价**：每克进货成本，用于利润统计
   - **库存**：初始库存克数（可选，后续通过进货补充）
   - **替代单位**（可选）：如大枣用"枚"、附子用"片"

### 4.3 替代计量单位

适合不以克计量的药材：

1. 展开「替代计量单位」
2. 选择单位（枚/个/片/升/合/等分）
3. 填写换算关系（如大枣：1枚 ≈ 3g）

> 设置后在开方页和模版中可使用替代单位输入，系统自动换算为克。

### 4.4 编辑药材

点击药材行右侧的 ✏️，修改后保存。库存变动会自动生成库存记录。

### 4.5 删除药材

点击 🗑 删除单个药材。已开药方和模版中的该药材名称会被保留（作为纯文本），不会丢失历史数据。

### 4.6 批量导入

支持 CSV 导入。详见 [数据导入导出](#9-数据导入导出)。

## 5. 药方记录与随访

路径：侧边栏 → 📋 药方

### 5.1 筛选与搜索

- **搜索框**：按病人姓名或药材名称搜索
- **按病人筛选**：选择特定病人，只显示该病人的药方

### 5.2 查看药方明细

点击药方行，弹出详情窗口，显示：
- 病人姓名、日期、药味数
- 每味药材的克数、单价、小计
- 药方总价

### 5.3 复制药方

在详情窗口中点击「📋 复制」，将药方内容（格式如「当归 10g、黄芪 15g…」）复制到剪贴板。可直接粘贴到微信/病历中。

### 5.4 再开药方

点击「🔄 再开」，跳转到开方页并自动填入历史药方的药材和克数，修改后可保存为新的药方。

### 5.5 删除药方

点击「🗑 删除」，确认后删除。**药材库存会自动退回**。

### 5.6 随访评价

在药方详情窗口底部，可添加随访记录：

1. 选择效果评价：痊愈 / 显效 / 有效 / 无效 / 加重
2. 可选填写用药反馈（如"服药后睡眠改善"）
3. 点击「保存随访」

随访记录会显示在药方详情中，也会在病人详情页的历史药方中展示。

### 5.7 清空全部药方

点击右上角「清空」按钮，确认后删除所有药方。**所有药材库存会自动退回**。

## 6. 药方模版

路径：侧边栏 → 📐 模版

### 6.1 什么是药方模版

药方模版存储一组药材组合和默认克数（如「补中益气汤」包含黄芪、党参、白术等 8 味药）。在开方时可一键加载模版，快速填入药材。

### 6.2 创建模版

**方式一：从开方页保存**
开方时将当前药方存为模版（见 [2.8](#28-保存为模版)）。

**方式二：手动创建**
1. 在模版页点击「➕ 新建」
2. 输入模版名称
3. 搜索添加药材，设置默认克数
4. 点击「创建模版」

### 6.3 编辑模版

点击模版右侧 ✏️ 按钮，可修改名称、增删药材、调整克数。

### 6.4 使用模版

**在开方页使用**：搜索模版名称或从热门列表点击 → 药材自动填入。

**也支持替代单位输入**：如果药材配置了替代单位（如大枣的"枚"），模版中可切换克/枚显示和输入。

### 6.5 删除模版

点击 🗑 删除。删除模版不影响已保存的药方。

### 6.6 批量导入

支持 CSV 导入。详见 [数据导入导出](#9-数据导入导出)。

## 7. 库存管理

路径：侧边栏 → 📦 库存

### 7.1 库存概览

「库存概览」标签页显示所有药材的当前库存量，低于 50g 的显示红色「不足」标签。

### 7.2 进货登记

1. 点击「➕ 进货」
2. 搜索并选择药材
3. 填写进货量（克）
4. 可选填写新成本价（会更新该药材的成本价）
5. 点击「确认进货」

> 保存后库存自动增加，并生成进货记录。

### 7.3 流水记录

「流水记录」标签页显示最近 100 条库存变动：
- 🟢 进货：手动进货或删除药方退回
- 🔴 扣减：保存药方时自动扣减

每条记录包含药材名称、变动克数、日期。

### 7.4 库存自动扣减

- **保存药方时**：药方中的所有药材库存自动扣减
- **删除药方时**：库存自动退回
- 库存可以降到负数（表示实际有库存但系统未录入），UI 上有低库存警告

## 8. 利润统计

路径：侧边栏 → 📊 统计

### 8.1 时间筛选

顶部标签可切换：全部 / 本月 / 本季度。

### 8.2 KPI 概览

四个卡片展示核心指标：

| 指标 | 说明 |
|------|------|
| 总营收 | 所有药方的售价合计 |
| 总成本 | 所有药方中的药材成本合计 |
| 总利润 | 营收 − 成本 |
| 药方数 | 筛选范围内的药方总数 |

### 8.3 药材维度排行

按利润降序排列每种药材的：
- 使用次数（出现在多少个药方中）
- 总营收贡献
- 总利润贡献

> 可快速识别高利润和常用药材，辅助定价决策。

### 8.4 数据导出

点击右上角「📥 导出」按钮，将所有数据（药材、病人、药方、模版、随访、库存记录）导出为 JSON 文件，可用于备份或迁移。

## 9. 数据导入导出

### 9.1 CSV 导入药材

**文件格式**：UTF-8 编码的 CSV

| 列名 | 必填 | 说明 |
|------|------|------|
| 名称 | ✅ | 药材中文名 |
| 售价 | — | 每克售价（元） |
| 成本价 | — | 每克成本价（元） |
| 单位 | — | 替代单位（枚/个/片/升/合） |
| 单位克数 | — | 1个替代单位 = ?克 |

示例：
```csv
名称,售价,成本价,单位,单位克数
当归,0.35,0.22
黄芪,0.28,0.15
党参,0.42,0.30
大枣,0.05,0.03,枚,3
```

> 表头支持中文和英文（name/sellPrice/costPrice/unit/unitGrams）。已存在的药材（按名称判断）会被跳过。

### 9.2 CSV 导入病人

| 列名 | 必填 | 说明 |
|------|------|------|
| 姓名 | ✅ | 病人姓名 |
| 性别 | — | 男/女 |
| 年龄 | — | 整数 |
| 电话 | — | 联系电话 |

示例：
```csv
姓名,性别,年龄,电话
张三,男,35,13800138001
李四,女,28
```

### 9.3 CSV 导入药方模版

| 列名 | 必填 | 说明 |
|------|------|------|
| 模版名称 | ✅ | 模版名称 |
| 药材 | ✅ | 药材列表，用 `|` 分隔 |

每种药材的格式：`药名 克数`（克数可省略单位 g），或 `药名 数量单位`（如 `大枣 12枚`）。

支持的单位：枚、个、片、升、合

示例：
```csv
模版名称,药材
补中益气汤,黄芪 15g|党参 10g|白术 10g|当归 8g|陈皮 6g|升麻 3g|柴胡 3g|甘草 5g
四君子汤,党参 12g|白术 12g|茯苓 12g|甘草 6g
桂枝汤,桂枝 10g|白芍 10g|甘草 6g|生姜 9g|大枣 12枚
```

> 模版中的药材必须在系统中已存在（按名称匹配），不存在的药材仅保留名称，克数按默认换算。

### 9.4 全量数据导出

在「统计」页点击「导出」，下载包含所有数据的 JSON 文件。JSON 中包含完整的药材、病人、药方、模版、随访和库存记录。

## 11. 常见问题

### Q: 开方时发现药材价格变了，会影响历史药方吗？

**不会。** 每个药方保存时都会快照当时的药材售价（价格快照），后续修改药材售价不影响已保存的药方。

### Q: 删除药材后，历史药方还能看到吗？

**能。** 删除药材时，药方中该药材的名称会作为纯文本保留，药方明细完整不丢失。但药材关联会被解除，无法跳转到药材详情。

### Q: 库存扣到负数了怎么办？

这是允许的——系统设计为"不因库存不足阻止开方"。常见原因：
1. 实际有库存但未在系统中录入（通过「进货登记」补充）
2. 直接从药材管理修改库存（自动生成调整记录）

### Q: 如何修改已保存的药方？

当前版本不支持直接编辑历史药方。操作方式：
1. 在药方记录中找到目标药方，点击「再开」
2. 修改药材或克数
3. 保存为新药方
4. 如需要，删除原药方（库存会自动退回并重新扣减）

### Q: 数据存储在哪里？

数据库文件为 `prisma/dev.db`（SQLite），存储在项目目录内。建议定期：
- 通过「统计 → 导出」备份数据
- 复制 `dev.db` 文件做完整备份

### Q: 如何迁移到新服务器？

1. 复制整个项目目录
2. 复制 `prisma/dev.db` 数据库文件
3. 配置 `.env` 环境变量
4. `npm install && npx prisma generate && npm run build && npm start`

### Q: CSV 导入失败怎么办？

- 确保文件编码为 **UTF-8**（不要用 GBK）
- 检查表头列名是否正确
- 用 Excel/Numbers 打开检查是否有格式错误
- 导入结果会显示成功/跳过/失败的数量
