# 百草计 — UI 问题分析与改进方案

> 日期：2026-07-10 | 状态：待实施

---

## 一、问题清单

### 1. 搜索框交互失效

**根因：** 开方页药材搜索采用 `Popover + Command` 双层结构——点击静态触发按钮后弹出浮层，浮层内才是真正的输入框。在手机端是"两步点击"，用户看到的是普通输入框外观，点击后却弹出另一个搜索面板，交互预期不符。

病人关联同样使用纯 `<Select>` 下拉，无搜索能力，病人多了完全不可用。

**修复方案：** 将两处都改为 **inline combobox**——输入框直接获焦，下方展开候选列表（最多 5 条），失焦收起，无二级弹层。

---

### 2. 库存与成本价逻辑冲突

**现状：** 成本价有两个入口同时可以修改：

| 入口 | 文件 | 行为 |
|------|------|------|
| 药材管理 → 添加/编辑表单 | `herbs/page.tsx` | 直接写入 `costPrice` |
| 库存管理 → 进货弹窗 | `inventory/page.tsx` | 进货后以 `unitPrice` 覆盖 `costPrice` |

用户不清楚哪个是"正式"入口，容易两处改了互相覆盖。

**修复方案：**
- 药材表单的成本价字段改为**只读 + 灰色**，显示说明文案"成本价通过进货自动更新"
- 进货弹窗是唯一修改成本价的入口，表单中明确标注"保存后将更新该药材成本价"
- 库存概览每行只展示 `stock(g)` 与低库存警告，不再展示 `costPrice`（成本价属于统计维度）

---

### 3. 病人管理问题

| # | 问题 | 影响 |
|---|------|------|
| 3-1 | 底部导航只有 3 个 Tab（开方/药方/库存），病人和药材无快捷入口 | 移动端难以发现 |
| 3-2 | 开方页选病人用纯 `<Select>`，无搜索 | 病人超过 10 人即不可用 |
| 3-3 | 药方历史页病人筛选同为 `<Select>` | 同上 |
| 3-4 | 病人详情页无"为此病人开方"快捷入口 | 需要返回再切换页面 |
| 3-5 | 删除用浏览器原生 `confirm()`，风格不一致 | PWA 内显示浏览器原生弹窗 |
| 3-6 | 病人搜索不支持拼音 | 中文姓名输入慢 |

**修复方案：**
- 底部导航扩至 5 个 Tab：**开方 / 药方 / 病人 / 药材 / 统计**
- 病人与药材搜索统一改为 inline combobox + 拼音支持
- 病人详情页顶部加"开新方"按钮，跳转开方页并预选该病人
- 删除确认改为 shadcn `AlertDialog`

---

### 4. 全局 UI 视觉（详见第二节线稿）

现有设计系统暖米黄底色偏"旧网页"风，草绿主色 `#2D6A3F` 过于低沉，浅绿选中背景 `#E8F0EA` 太淡、无存在感。详见"竹素"配色升级方案（`design.md`）。
