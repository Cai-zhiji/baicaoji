# 百草计 — 产品需求文档

> 版本：v1.0 | 日期：2026-07-09 | 状态：开发中（MVP 核心功能已完成，随访/CSV/PWA 待开发）

---

## 1. 产品概述

### 1.1 产品定位

**百草计**是一款面向中医从业者的轻量级 Web 应用，核心解决两个问题：

1. **快速计价**：开方时逐味添加药材、填入克数，药方总价实时计算。
2. **诊务积累**：长期记录病人、药方、随访反馈，形成个人诊疗知识库。

支持 PWA，可安装到电脑桌面或手机桌面，像原生 App 一样使用。

### 1.2 目标用户

单人使用 —— 一位中医从业者。无多用户、无权限体系、无社交功能。

### 1.3 使用场景

| 场景 | 描述 |
|------|------|
| 开方计价 | 病人就诊，逐味搜药、加药、填克数，实时看到药方总价，保存后关联病人。 |
| 复诊参考 | 查看某位病人的历史药方和随访记录，了解过往用药效果。 |
| 模版复用 | 加载常用药方模版，填入克数后快速完成计价。 |
| 库存管理 | 进货登记，开方自动扣减库存，成本价联动更新。 |
| 利润统计 | 查看某段时间内的营收和利润（售价 - 成本）。 |
| 数据维护 | 空闲时补充随访记录、批量导入药材/模版、导出数据备份。 |

## 2. 功能需求

### 2.1 药材管理

**字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 名称 | 文本 | 中药材中文名，如"当归" |
| 拼音 | 文本 | 系统根据名称自动生成，如 "danggui" |
| 售价 | 数字 | 每克售价（元），精确到分 ¥0.00 |
| 成本价 | 数字 | 每克进货成本（元），精确到分 ¥0.00 |

**功能：**
- 药材的增、删、改。
- **拼音首字母 + 汉字搜索**：输入 "dg" 或 "当" 都能搜到"当归"。
- **CSV 批量导入**：支持上传包含（名称、售价、成本价）的 CSV 文件，一键导入药材库。

### 2.2 药方计价（核心流程）

**操作流程：**
1. 选择病人（或新建病人）。
2. 搜索药材 → 选中加入药方 → 填入克数（支持小数）。
3. 继续添加下一味药材，重复直到方子完整。
4. **药方总价实时显示**：总价 = Σ(每味药材售价 × 克数)，精确到分。
5. 保存药方，关联病人。

**药方明细行：**

| 字段 | 说明 |
|------|------|
| 药材名称 | 从药材库选择 |
| 克数 | 支持小数（如 3.5g） |
| 单价 | 自动带入药材当前售价 |
| 小计 | 单价 × 克数 |

**总价** = 所有明细行小计之和。

### 2.3 药方模版

**定义：** 仅包含药材组合（不含克数）的可复用模版。

**来源：**
- **手动保存**：从已开药方中点击"保存为模版"，提取其中的药材列表（丢弃克数）。
- **批量导入**：通过 CSV 批量导入模版（模版名称 + 药材列表）。

**使用方式：**
- 开方时加载模版 → 药材列表自动填入药方 → 逐味填克数 → 计价 → 保存。

### 2.4 病人管理

**字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 姓名 | 文本 | **唯一标识**，同名自动归入同一档案 |
| 性别 | 枚举 | 男 / 女 |
| 年龄 | 数字 | 或出生年份 |
| 联系电话 | 文本 | 可选 |

**同名规则：** 系统以姓名为唯一标识。输入相同姓名时，自动归档到已有病人档案，历史药方合并展示。

### 2.5 随访记录

**关联对象：** 具体的某一次药方（而非病人）。

**字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 效果评价 | 枚举（五档） | 痊愈 / 显效 / 有效 / 无效 / 加重 |
| 文字描述 | 长文本 | 自由填写 |
| 记录时间 | 日期时间 | 系统自动生成 |

**使用场景：** 空闲时补充录入，记录病人用药后的反馈。

### 2.6 库存管理

**进货：**
- 手动录入：选择药材 → 填入进货克数 → 填入进价。
- 进价自动更新该药材的**成本价**字段。

**扣减：**
- 开方保存时，系统自动扣减对应药材的库存克数。

**提醒：**
- 库存不足时（低于某阈值或开方克数超过库存），给出提示。

**单位统一：** 所有库存操作单位均为**克**。进货时如需按公斤录入，用户自行换算（1 kg = 1000 g）。

### 2.7 利润统计

- **利润 = 药方总售价 - 药方总成本**（成本 = Σ(每味药材成本价 × 克数)）。
- 支持按时间段（本月、本季度、自定义）查看累计利润。
- 前端开方时不展示成本和利润，仅展示药方总售价；利润数据在后端统计页面查看。

### 2.8 数据安全

- **数据导出备份**：系统提供"导出数据"功能，下载包含所有数据的文件。用户定期手动备份到本地。

## 3. 非功能需求

### 3.1 PWA

- 支持安装到桌面（电脑）/ 主屏幕（手机）。
- 离线时展示应用壳（基本 UI），数据操作需联网。
- 应用图标、全屏体验、无浏览器地址栏。

### 3.2 响应式设计

- 电脑端（宽屏）：药材搜索区和药方编辑区可并排展示。
- 手机端（窄屏）：上下堆叠布局，拇指友好操作。

### 3.3 性能

- 药材搜索实时响应（本地过滤或后端轻量查询）。
- 药方总价实时计算，无需点击"计算"按钮。

## 5. 数据模型（概要）

> 实际实现 8 张表（比原规划多出 TemplateItem 独立表，StockRecord 为新增流水表）

```
药材 (Herb)
  ├── id
  ├── name           # 名称
  ├── pinyin         # 拼音（自动生成）
  ├── sellPrice      # 售价/克
  ├── costPrice      # 成本/克
  ├── stock          # 库存克数
  └── ...

病人 (Patient)
  ├── id
  ├── name           # 姓名（唯一）
  ├── gender         # 性别
  ├── age            # 年龄
  ├── phone          # 电话
  └── ...

药方 (Prescription)
  ├── id
  ├── patientId      → Patient
  ├── totalPrice     # 总售价
  ├── totalCost      # 总成本
  ├── createdAt
  └── ...

药方明细 (PrescriptionItem)
  ├── id
  ├── prescriptionId → Prescription
  ├── herbId         → Herb
  ├── grams          # 克数
  ├── unitPrice      # 开方时单价（快照）
  └── ...

药方模版 (Template)
  ├── id
  ├── name           # 模版名称
  └── ...

模版明细 (TemplateItem)
  ├── id
  ├── templateId     → Template
  ├── herbId         → Herb
  └── ...

随访记录 (FollowUp)
  ├── id
  ├── prescriptionId → Prescription
  ├── evaluation     # 五档评价
  ├── note           # 文字描述
  ├── createdAt      # 自动时间
  └── ...

库存记录 (StockRecord) ✨ 已实现
  ├── id
  ├── herbId         → Herb
  ├── type           # "in"（进货） / "out"（扣减）
  ├── grams          # 克数
  ├── unitPrice      # 进货单价（仅进货时）
  ├── createdAt
  └── ...
```

## 8. 附录：决策记录

| 决策点 | 选择 | 原因 |
|--------|------|------|
| 用户模式 | 单人使用 | 唯一用户，无需多账号 |
| 数据存储 | SQLite 数据库 | 轻量、免维护、单文件、备份简单 |
| 框架 | Next.js | React 生态、全栈能力、PWA 成熟 |
| UI | Tailwind + shadcn/ui | 现代简洁、响应式、组件丰富 |
| 搜索方式 | 拼音首字母 + 汉字 | 中文输入慢时的快捷方案 |
| 病人标识 | 姓名（同名归并） | 单人场景下最自然的方式 |
| 克数精度 | 允许小数 | 实际开方存在 3.5g 等用法 |
| 价格精度 | 精确到分（¥0.00） | 计价场景的标准精度 |
| 模版 | 仅药材组合、不含克数 | 加载后灵活填克数 |
| 随访评价 | 五档（痊愈→加重） | 满足临床评价粒度 |
| 离线策略 | 在线为主、PWA 仅安装体验 | 数据存后端数据库，离线无意义 |
| 备份 | 手动导出 | 用户主动触发，简化系统 |
| 部署 | 阿里云轻量服务器 + 域名 | 国内访问快、成本低 |
