# 百草计 — 领域词汇表

> 最后更新：2026-07-09

## 核心概念

### 药材（Herb）
一味中药材。系统中最基础的计价单元。
- **名称**：药材的中文名，如"当归""黄芪"。
- **拼音**：系统根据名称自动生成，用于拼音首字母搜索（输入 "dg" 搜到"当归"）。
- **售价**：每克售价，单位 元/克，精确到分（¥0.00）。
- **成本价**：每克进货成本，单位 元/克，精确到分（¥0.00）。用于利润统计。

### 药方（Prescription）
一次开方记录。由多味药材组成，每味有各自的克数。药方总价 = Σ(每味药材售价 × 克数)，精确到分。药方必须关联一位病人，不可匿名开方。

### 药材明细（Prescription Item）
药方中的一行：{药材, 克数}。克数支持小数（如 3.5g）。开方时快照药材当时的单价。

### 药方模版（Prescription Template）
仅包含药材组合、不含克数的复用模版。加载模版后，用户逐味填入克数再计价。来源：
1. 从已开药方手动保存为模版（自动丢弃克数）。
2. CSV 批量导入创建（模版名称 + 药材列表）。

### 病人（Patient）
以**姓名为唯一标识**的就诊人。同名自动归入同一档案，合并历史药方记录。
- 字段：姓名、性别、年龄、联系电话。

### 随访记录（Follow-up）
关联到某一次**具体药方**的用药反馈，而非病人整体。
- **效果评价**（五档）：痊愈 / 显效 / 有效 / 无效 / 加重。以勾选方式选择。
- **文字描述**：自由文本。
- **记录时间**：系统自动生成。

### 库存（Inventory）
药材的库存管理，所有操作单位为克。
- **进货**：手动录入（药材、克数、进价），进价自动更新该药材的成本价。
- **扣减**：开方保存后自动扣减对应药材库存克数。
- **提醒**：库存不足时提示。

### 利润（Profit）
统计维度：利润 = 药方总售价 - 药方总成本（成本 = Σ(每味药材成本价 × 克数)）。在统计页面查看，开方时前端不展示。

## 用户

系统为**单人使用**。一位中医从业者，所有数据自管。通过简单密码登录保护。

## 边界外

以下概念不属于本系统范围：
- 多用户 / 多账号 / 权限控制
- 在线支付 / 收费管理
- 预约挂号 / 排班
- 医保对接
- 药材配伍禁忌自动检测
- 药方打印（V2.0 考虑）
